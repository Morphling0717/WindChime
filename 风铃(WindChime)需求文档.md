🎐 「风铃 (WindChime)」专属提问箱组件 (单主播内嵌版) 功能需求文档 (PRD)

产品定位：一款专为单一虚拟主播（VTuber/创作者）个人网站定制的轻量级匿名提问组件。通过前后端分离的设计，前端提供高度可复用的 React 模板组件嵌入个人站接收粉丝纯粹的留言，后端提供高效的消息管理、一键删除/拉黑功能，以及便于在直播间展示的互动工具。风铃随风而动，倾听最真实的声音。

一、 核心业务闭环 (Core Loop)

粉丝/访客：访问主播的个人网站 -> 在提问箱组件中输入文字并发送。

主播 (唯一管理员)：登录专属的隐藏管理后台 -> 查看所有收到的原生态留言 -> 决定保留、回复或直接删除/拉黑。

展示与互动：主播挑选喜欢的留言，生成精美图片导出至社交平台，或直接推送到 B站直播姬/OBS 等直播画面进行互动。

二、 详细功能模块拆解（从大到小）

1. 前端内嵌发送组件 (Embedded Sender Widget - 模板化设计)

该模块将作为一个高内聚的 React 组件模板直接嵌入主播现有的 Next.js 网站项目中。极度注重可扩展性与换肤能力。

UI/UX 模板化展示

主题无缝切换：基于 Tailwind CSS，支持通过 Props 传入不同的 themeColor 类名（如主色调、背景、高亮色），瞬间适配不同虚拟主播网站的视觉风格。

默认质感：提供开箱即用的轻盈玻璃拟态（Glassmorphism）质感模板。

状态展示：显示“当前风铃状态”（如：迎风飘摇中 / 休息避风中）。

自定义主播寄语：输入框上方的简短提示语支持外部传入（如：“把你想说的话，挂在风铃上吧~”）。

消息输入框

多行文本输入，支持字数统计（如最大 1000 字）。

发送状态反馈

发送按钮加载动画（防连点保护）。

发送成功后的特效：风铃轻轻摇曳的 CSS 动画（可配置是否伴随音效）。

成功提示语（“你的心意已随风传达啦！”）。

2. 基础安全与防洪机制 (Security & Anti-Spam)

虽然去除了内容过滤，但仍需防止恶意脚本造成的服务器瘫痪或数据库灌水。

前端防刷：基于浏览器 Cookie/LocalStorage 的冷却限制（如 1 分钟内最多发 3 条）。

后端频率限制 (Rate Limiting)：基于 IP 的请求频率限制，防止 Next.js API Routes 被恶意爆破。

人机验证：集成隐形验证码（如 Cloudflare Turnstile），阻挡自动化机器人发帖。

3. 主播专属管理后台 (Admin Dashboard - 核心管理区)

独立的隐藏路由（如 /admin-box），由于没有 AI 过滤，此处的高效处理体验至关重要。

安全认证

单账号密码登录 / 密钥 Token 登录。

收件箱管理 (Inbox Management)

分类标签：全部信笺 (All)、未读 (Unread)、红心收藏 (Favorited)、已回复 (Replied)。

列表展示：按时间倒序排列，显示留言内容片段、接收时间、隐式发送者标识（如 User-1A2B，方便识别是否是同一个人持续刷屏）。

高效操作 (Action Bar)

一键删除：遇到不喜欢的内容，直接点击删除（数据库逻辑软删除）。

批量管理：支持多选卡片进行批量删除、批量已读。

拉黑发送者 (Block)：极其重要的防御手段。通过记录发送者的隐式哈希（IP + UserAgent 生成的唯一标识）写入数据库黑名单，该标识未来发送的所有内容将直接被 API 丢弃。

撤销拉黑：提供黑名单列表面板，支持解除误封。

4. 直播与社交分享工具 (Broadcast & Share Tools)

专属链接与二维码引流 (Link & QR Code Sharing)

二维码生成器：在管理后台提供一键生成功能，自动将风铃组件所在的网页 URL 转换为高清二维码。

个性化定制：支持在二维码中心嵌入主播的头像或专属 Logo，并自动匹配当前项目的 themeColor 作为二维码的前景色。

宣传海报下载：不仅生成单张二维码，还可以生成带有“扫码给我留言”等引导文案和精美背景的宣传海报（PNG 格式），方便主播直接发布到 B站动态、Twitter 或朋友圈。

图片生成器 (Image Generator - 回复粉丝用)

将选中的留言文字自动排版在一张美观的风铃信笺图片上。支持长图自适应与局部鼠标划选打码。一键下载 PNG。

直播间互动模式 (直播姬 / OBS 兼容)

提供一个透明背景的独立路由作为浏览器源 (Browser Source)，无论是 B站直播姬 还是 OBS 均可通过“添加网页/浏览器”功能直接无缝接入。

主播在后台点击“展示”时，通过状态同步触发直播软件画面中弹出该消息的精美气泡动画。

5. 运营设置与状态管理 (Settings & Status)

组件开关与文案

一键开启/关闭提问功能（数据库状态位控制）。

支持自定义关闭时的前端提示语，让粉丝感受到更多人情味。

三、 技术架构规范 (针对多项目模板库)

既然目标是作为一套可供多个虚拟主播项目复用的模板集，底层架构将全面拥抱现代全栈标准：

统一核心技术栈：

框架：React + Next.js (App Router / Pages Router 皆可兼容)。

样式：Tailwind CSS (极致的组件化样式封装，便于换肤)。

类型系统：TypeScript (严格的接口定义，保证跨项目引入组件时的类型安全和代码提示)。

数据存储 (SQL 数据库)：

采用关系型数据库进行稳健的数据持久化（PostgreSQL / MySQL / SQLite）。

ORM 推荐：使用 Prisma 或 Drizzle ORM。这两者与 TypeScript 和 Next.js 契合度极高，能够通过统一的 Schema 管理所有主播的留言数据和黑名单记录。

对于多项目管理，可以选择每个主播分配独立的数据库，或者采用多租户（Multi-tenant）结构在同一个数据库中通过 projectId 区分数据。

后端接口 (API Routes)：

直接利用 Next.js 自带的 API Routes (或 Server Actions) 处理前端表单提交、存入 SQL 数据库，实现前后端同构，免去了独立部署后端服务器的麻烦。

状态同步 (针对直播浏览器源)：

为了轻量化，推荐使用 Server-Sent Events (SSE) 或基于数据库的短轮询 (SWR / React Query) 来实现后台操作到直播渲染页面（直播姬/OBS）的即时同步，避免引入沉重的 WebSocket 依赖。
